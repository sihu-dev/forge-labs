# Think Deep - 확장 사고 모드

복잡한 문제에 대해 깊이 있는 분석과 계획을 수행합니다.

## 사용법
```
/think-deep [문제 또는 기능 설명]
```

## 트리거 키워드
- `think hard` - 심층 분석 모드
- `ultrathink` - 최대 깊이 분석

## 프로세스

### Phase 1: 문제 이해 (코드 작성 금지)
1. 요구사항 완전히 파악
2. 기존 코드베이스 분석
3. 제약 조건 식별
4. 엣지 케이스 나열

### Phase 2: 설계 탐색
1. 최소 3가지 접근 방식 제시
2. 각 방식의 장단점 분석
3. 기술적 트레이드오프 평가
4. 최적 솔루션 선택 및 근거

### Phase 3: 구현 계획
1. 단계별 구현 로드맵
2. 파일 변경 목록
3. 테스트 전략
4. 롤백 계획

### Phase 4: 실행 (승인 후)
1. 계획에 따른 구현
2. 각 단계 완료 후 검증
3. 테스트 실행
4. 문서화
