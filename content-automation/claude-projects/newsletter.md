## Subject Line Formulas

### Formula 1: Curiosity Gap
```
The {tool/method} I can't stop using
```

**Examples**:
- The automation hack I've been gatekeeping
- The pricing strategy nobody talks about
- The Claude prompt that 10x'd my writing

### Formula 2: FOMO (Fear of Missing Out)
```
You're missing out on {benefit}
```

**Examples**:
- You're missing out on free traffic (here's how to fix it)
- Your competitors are using this (and you're not)
- While you're {doing X}, others are {doing Y}

### Formula 3: Urgency
```
{X} hours left to {action}
```

**Examples**:
- 24 hours left to grab this template
- Last chance for early access
- Closing registration in 3 hours

### Formula 4: Personalization
```
{Name}, your {thing} is {state}
```

**Examples**:
- Sarah, your content system is broken (here's why)
- Your newsletter could be making 10x more
- {Name}, I built this for you

### Formula 5: Question
```
Still {struggling with X}?
```

**Examples**:
- Still writing content manually?
- Are you making this SEO mistake?
- What if you could {benefit}?

### Formula 6: Number + Benefit
```
{Number} {things} to {achieve outcome}
```

**Examples**:
- 3 automations that saved me 20 hours this week
- 5 newsletters every founder should subscribe to
- 7 Claude prompts for instant content ideas

### Formula 7: Preview Text Combo
```
Subject: {Curiosity}
Preview: {Additional context or benefit}
```

**Examples**:
- Subject: I broke something
  Preview: ...and it 3x'd my revenue

- Subject: This sounds insane, but...
  Preview: ...it cut my content time by 80%

## A/B Testing Strategy

### What to Test

**Priority 1: Subject Lines**
```yaml
Test_1:
  variant_a: "How to automate content creation"
  variant_b: "The content automation stack I use"
  hypothesis: "Specific ('I use') beats generic ('How to')"
  metric: "Open rate"
```

**Priority 2: CTA Placement**
```yaml
Test_2:
  variant_a: "CTA at end of email"
  variant_b: "CTA mid-email + end"
  hypothesis: "Multiple CTAs increase clicks"
  metric: "Click-through rate"
```

**Priority 3: Email Length**
```yaml
Test_3:
  variant_a: "500 words"
  variant_b: "250 words (trimmed)"
  hypothesis: "Shorter = higher engagement"
  metric: "Read time + CTR"
```

### Testing Calendar

**Week 1**: Subject line (curiosity vs. benefit)
**Week 2**: Preview text (with vs. without)
**Week 3**: CTA copy ("Read more" vs. "Get the guide")
**Week 4**: Email length (long vs. short)

**Rotate**: Don't test same variable back-to-back

### Winning Criteria

**Open Rate**:
- Baseline: 30%
- Winner: +5% absolute improvement (e.g., 30% ‚Üí 35%)

**Click-Through Rate**:
- Baseline: 5%
- Winner: +2% absolute improvement (e.g., 5% ‚Üí 7%)

**Implement winner**: If confident after 2 consecutive tests

## Email Design & Formatting

### Plain Text vs. HTML

**Use Plain Text**:
- Personal newsletters from founder
- 1-on-1 feeling
- Higher deliverability
- Mobile-friendly

**Use HTML**:
- Product announcements (need images/buttons)
- Curated digests (better visual hierarchy)
- Newsletters with ads/sponsors

**Hybrid Approach** (Recommended):
- Mostly plain text
- Minimal styling (bold, italic, links)
- 1-2 images max
- HTML for CTA button only

### Formatting Rules

**Line Length**: 60-70 characters (improves readability)

**Paragraphs**: 2-3 sentences max

**White Space**: Double line breaks between sections

**Bold**: For subheadings and key phrases

**Links**:
```markdown
‚ùå Click here: https://example.com/very-long-url
‚úÖ [Descriptive anchor text](https://example.com/url)
```

**Code Blocks**:
```
Use triple backticks for code

```javascript
const example = 'formatted code';
```
```

**Lists**:
```markdown
Numbered (for steps):
1. First action
2. Second action

Bullet (for options):
- Option A
- Option B
```

## Metrics & KPIs

### Primary Metrics

**Open Rate**:
- **Target**: 30-40% (good), 40-50% (great)
- **Factors**: Subject line, send time, sender name, list hygiene

**Click-Through Rate (CTR)**:
- **Target**: 5-10% (good), 10-15% (great)
- **Factors**: Content quality, CTA placement, link relevance

**Unsubscribe Rate**:
- **Target**: <0.5% per send
- **Acceptable**: <1%
- **Red Flag**: >2% (poor targeting or frequency)

**Reply Rate**:
- **Target**: >1% (indicates strong engagement)
- **How**: Ask questions, encourage replies

### Secondary Metrics

**Read Time**:
- Track with tools like Mailchimp or ConvertKit
- Aim for >50% email read (scroll to bottom)

**Conversion Rate**:
- % of clicks that lead to desired action (purchase, signup)
- Depends on offer and audience

**Deliverability**:
- Inbox placement rate (avoid spam folder)
- Monitor with tools like GlockApps

**List Growth Rate**:
- New subscribers - unsubscribes / total subscribers
- Target: +5-10% per month

### Analytics Dashboard

Track weekly in Notion or spreadsheet:

## Deliverability Best Practices

### Avoid Spam Filters

**Subject Line Red Flags**:
- ‚ùå ALL CAPS
- ‚ùå Excessive punctuation!!!!!
- ‚ùå $$$$ or FREE
- ‚ùå RE: or FWD: (unless actually reply)

**Content Red Flags**:
- ‚ùå Too many links (>5)
- ‚ùå Large images (>500KB)
- ‚ùå Spammy words: "click here", "limited time", "act now"

**Technical Setup**:
- ‚úÖ SPF record configured
- ‚úÖ DKIM signature enabled
- ‚úÖ DMARC policy set
- ‚úÖ Custom sending domain (mail.yourdomain.com)

### List Hygiene

**Clean List Every Quarter**:
- Remove hard bounces immediately
- Remove soft bounces after 3 attempts
- Remove inactive (no opens in 90 days) after re-engagement campaign

**Double Opt-In**:
- Require email confirmation
- Ensures real, engaged subscribers
- Improves deliverability

## Emergency Protocols

### If Send Goes Wrong

**Typo in subject/body**:
- Send follow-up: "Oops, typo in the last email..." (humanizes you)
- Don't resend unless critical error

**Broken link**:
- Send immediate correction email
- Subject: "Quick fix: {original subject}"

**Wrong segment sent**:
- Apologize quickly
- If possible, resend to correct segment with note

### Unsubscribe Spike

**If >2% unsubscribe rate**:
1. Review content - was it off-brand or too salesy?
2. Check frequency - did you send too often?
3. Survey leavers: "Sorry to see you go - what went wrong?"
