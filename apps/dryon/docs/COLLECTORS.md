# 공고 수집 소스 (Data Sources)

## 개요

Hyein Agent는 **16개 이상의 공고 소스**에서 정부지원사업, 창업지원, R&D 과제 등의 공고를 자동으로 수집합니다.

## 수집 소스 목록

### 1. 정부기관 (Government Agencies)

#### 기업마당 (Bizinfo) ⭐⭐⭐
- **운영**: 중소벤처기업부
- **URL**: https://www.bizinfo.go.kr
- **설명**: 정부 지원사업 통합 포털
- **API**: 공공데이터포털 API 사용
- **수집 항목**: 중소기업, 벤처, 창업 지원사업 전체
- **우선순위**: 최우선 (10/10)
- **상태**: ✅ 활성화

**특징:**
- 중소벤처기업부 산하 모든 지원사업 통합
- 가장 많은 공고 보유
- API 안정성 높음

## 수집 방식

### API 기반 수집
- **소스**: 기업마당, K-Startup, (기술보증기금 API)
- **장점**:
  - 안정적인 데이터
  - 구조화된 정보
  - 빠른 속도
- **단점**:
  - API 키 필요
  - 승인 필요 (1-2일)

### 웹 크롤링 기반 수집
- **소스**: 나머지 모든 소스
- **장점**:
  - API 키 불필요
  - 즉시 사용 가능
  - 유연한 대응
- **단점**:
  - 웹사이트 구조 변경시 수정 필요
  - API 대비 느림

## 수집 프로세스

### 1. 병렬 수집
```
┌─────────────┐
│ 기업마당    │ ────┐
└─────────────┘     │
┌─────────────┐     │
│ K-Startup   │ ────┤
└─────────────┘     │
┌─────────────┐     │    ┌──────────────┐
│ 창업진흥원  │ ────┼───>│ CollectorManager │
└─────────────┘     │    └──────────────┘
┌─────────────┐     │           │
│ NIPA        │ ────┤           │
└─────────────┘     │           v
┌─────────────┐     │    ┌──────────────┐
│ ...         │ ────┘    │ 중복 제거    │
└─────────────┘          └──────────────┘
  (동시 5개까지)                │
                                v
                         ┌──────────────┐
                         │ AI 분석      │
                         └──────────────┘
```

### 2. 우선순위 기반 수집
- 우선순위 높은 소스부터 수집
- 실패해도 다른 소스 계속 수집
- 부분 실패 허용

### 3. Rate Limiting
- 동시 최대 5개 소스
- 초당 최대 10개 요청
- 과부하 방지

## 중복 제거

### 중복 판단 기준
```typescript
key = `${title.trim()}_${agency.trim()}`
```

- 제목 + 주관기관 조합으로 판단
- 대소문자 구분 안함
- 공백 제거 후 비교

### 예시
```
✅ 동일 공고로 판단:
- "2025 예비창업패키지" (기업마당)
- "2025 예비창업패키지" (K-Startup)

❌ 다른 공고로 판단:
- "2025 예비창업패키지" (서울시)
- "2025 예비창업패키지" (경기도)
```

## 수집기 관리

### 수집기 활성화/비활성화

```typescript
import { collectorManager } from './services/collectors/collector-manager';

// 특정 수집기 비활성화
collectorManager.setCollectorEnabled('busan', false);

// 수집기 목록 확인
const collectors = collectorManager.listCollectors();
```

### 카테고리별 수집

```typescript
// 정부기관만 수집
const announcements = await collectorManager.collectByCategory('government');

// 지자체만 수집
const announcements = await collectorManager.collectByCategory('local');
```

## 확장 가이드

### 새로운 수집기 추가

1. **수집기 클래스 생성**

```typescript
// src/services/collectors/new-source-collector.ts
export class NewSourceCollector {
  async collect(keywords?: string[]): Promise<Announcement[]> {
    // 수집 로직 구현
  }
}
```

2. **CollectorManager에 등록**

```typescript
// collector-manager.ts에 추가
{
  id: 'new-source',
  name: '새로운 소스',
  description: '설명',
  enabled: true,
  priority: 7,
  instance: new NewSourceCollector(),
  category: 'platform',
}
```

3. **즉시 사용 가능!**

## 문제 해결

### 수집 실패시
```
❌ NIPA 공고 수집 실패
→ 해당 소스만 실패, 다른 소스는 정상 수집
```

### 웹사이트 구조 변경
- 크롤링 기반 수집기는 웹사이트 구조 변경시 수정 필요
- GitHub Issues에 보고
- 또는 직접 수정 후 PR

### API Rate Limit
```env
# .env에서 조정
PUBLIC_API_RATE_LIMIT_PER_MINUTE=50
```
